# Copilot Automation Guidelines

Instructions for AI agents to assist in maintaining and organizing this repository.

## Primary Automation Tasks

### 1. Content Analysis and Organization
- **Analyze `raw-drops.md`**: Review new entries for categorization opportunities
- **Pattern Recognition**: Identify recurring themes and instruction patterns
- **Quality Assessment**: Evaluate instructions for clarity, specificity, and actionability
- **Consolidation Opportunities**: Detect similar or redundant instructions across files

### 2. Content Promotion Workflow
- **Promotion Criteria**: Instructions must be tested, specific, and reusable
- **Target Organization**: Move qualified content from `collected/` to appropriate `cleaned/` files
- **Template Creation**: Convert successful patterns into reusable templates
- **Cross-Reference Building**: Create connections between related instruction patterns

### 3. Quality Enhancement Automation
- **Clarity Improvement**: Suggest specific language improvements for ambiguous instructions
- **Context Addition**: Recommend additional context for complex instructions
- **Example Enhancement**: Add real-world examples and anti-patterns where helpful
- **Safety Review**: Ensure all promoted instructions include appropriate safety considerations

## AI Agent Integration Strategies

### Agent Mode Collaboration
**Use Case**: Real-time instruction refinement and testing
- **Prototype Review**: Test instruction effectiveness in real scenarios
- **Interactive Refinement**: Iterate on instruction clarity and specificity
- **Context Validation**: Verify instructions work across different project contexts
- **Quality Assurance**: Real-time feedback on instruction effectiveness

### Coding Agent Implementation
**Use Case**: Systematic repository maintenance tasks
- **Batch Processing**: Organize large volumes of raw content systematically
- **Consistency Enforcement**: Apply formatting standards across all files
- **Template Application**: Implement standardized instruction patterns
- **Documentation Updates**: Maintain current repository structure documentation

## Automation Workflow Patterns

### Pattern 1: Rapid Collection → Systematic Organization
1. **Agent Mode**: Quickly capture and add new instruction insights to `raw-drops.md`
2. **Coding Agent**: Systematically categorize and organize accumulated content
3. **Agent Mode**: Review and refine organized content for promotion
4. **Coding Agent**: Implement promotion to `cleaned/` directory with proper formatting

### Pattern 2: Quality-Driven Curation
1. **Coding Agent**: Analyze `collected/` content for promotion candidates
2. **Agent Mode**: Test instruction effectiveness and clarity
3. **Agent Mode**: Refine instructions based on testing feedback
4. **Coding Agent**: Promote refined instructions with proper categorization

### Pattern 3: Template Development
1. **Agent Mode**: Identify successful instruction patterns from examples
2. **Agent Mode**: Draft template structure with placeholders and guidelines
3. **Coding Agent**: Implement template in `prompt-templates.md` with proper formatting
4. **Agent Mode**: Test template effectiveness with real scenarios

## Quality Assurance Automation

### Automated Checks
- **Instruction Specificity**: Flag vague or ambiguous instructions for improvement
- **Context Completeness**: Identify instructions lacking necessary context
- **Safety Compliance**: Ensure all instructions include appropriate safety considerations
- **Consistency Validation**: Check formatting and structure consistency across files

### Testing Protocols
- **Effectiveness Testing**: Validate instructions produce expected outcomes
- **Context Testing**: Verify instructions work across different project types
- **Clarity Testing**: Ensure instructions are understandable without additional context
- **Safety Testing**: Confirm instructions don't introduce security or quality risks

## Repository Maintenance Automation

### File Structure Maintenance
- **Structure Validation**: Ensure repository follows documented organization patterns
- **Link Verification**: Check internal references and cross-links remain valid
- **Documentation Currency**: Keep README and structure documentation current
- **Format Consistency**: Maintain consistent markdown formatting across all files

### Content Lifecycle Management
- **Promotion Tracking**: Monitor content movement from raw to cleaned state
- **Usage Analytics**: Track which instructions are most frequently referenced
- **Update Scheduling**: Regular review cycles for instruction currency and relevance
- **Archive Management**: Handle outdated or superseded instruction patterns
